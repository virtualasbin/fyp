AT3 Testing Evidence
Student Task Management Web Application

Purpose
This testing evidence file is prepared for the AT3 video demonstration and is aligned with the AT2 requirements.

Demo Account
Email: demo@studenttasks.example.com
Password: Password123

How To Run
Option 1:
python -m pip install -r requirements.txt
python seed_demo.py --reset-demo
python -m uvicorn app.main:app --host 127.0.0.1 --port 8004

Option 2:
.\scripts\run_demo.ps1

Note:
The scripts\run_demo.ps1 script automatically checks whether the chosen port is already in use.
If the preferred port is busy, it starts the application on the next available port automatically.

Browser URL
Use the local URL shown by run_demo.ps1, for example:
http://127.0.0.1:8003

Functional Testing

Test 1: User Registration
Requirement: User registration
Steps:
1. Open the application.
2. Click Register.
3. Enter name, email, and password.
4. Submit the form.
Expected Result:
The account is created successfully and the user is logged in.

Test 2: User Login Success
Requirement: User login
Steps:
1. Open the login form.
2. Enter valid credentials.
3. Click Login.
Expected Result:
The dashboard opens and user details are shown.

Test 3: User Login Error
Requirement: Validation and secure authentication
Steps:
1. Open the login form.
2. Enter an invalid email or wrong password.
3. Click Login.
Expected Result:
An error message is shown and login is rejected.

Test 4: Task Creation
Requirement: Task creation, deadline assignment, priority tagging, reminder creation
Steps:
1. Enter a task title.
2. Enter description.
3. Select due date.
4. Select priority.
5. Select status.
6. Select module or category.
7. Add reminder time.
8. Click Save Task.
Expected Result:
The task is saved and appears in the task list and dashboard.

Test 5: Task Creation Validation
Requirement: Data validation
Steps:
1. Leave the title blank.
2. Try to submit the task form.
Expected Result:
The system shows a validation error near the title field.

Test 6: Invalid Reminder Validation
Requirement: Data validation
Steps:
1. Choose a reminder time after the due date.
2. Submit the task form.
Expected Result:
The system shows an error near the reminder field explaining the problem.

Test 7: Task Editing
Requirement: Task editing
Steps:
1. Select Edit on an existing task.
2. Change one or more fields.
3. Save the task.
Expected Result:
The updated values are shown in the task list.

Test 8: Task Completion
Requirement: Task tracking and prioritisation
Steps:
1. Select Mark Complete on an active task.
Expected Result:
The task status changes to completed and dashboard completion rate updates.

Test 9: Task Deletion
Requirement: Task deletion
Steps:
1. Select Delete on an existing task.
Expected Result:
The task is removed from the list.

Test 10: Category Creation
Requirement: Task categorisation
Steps:
1. Enter a new module name in the category section.
2. Click Add Module.
Expected Result:
The new category appears in the module list and task form.

Test 11: Task Filtering
Requirement: Task filtering
Steps:
1. Use the module, priority, or status filter.
Expected Result:
Only matching tasks are shown.

Test 12: Task Sorting
Requirement: Task sorting
Steps:
1. Change the sort option to due date, priority, or title.
Expected Result:
The task list order updates correctly.

Test 13: Dashboard Analytics
Requirement: Dashboard display, overdue task counts, completion rates, upcoming deadlines
Steps:
1. Log in using the demo account.
2. View the dashboard cards.
Expected Result:
Today's tasks, upcoming deadlines, overdue count, and completion rate are displayed.

Test 14: Settings Section
Requirement: Settings, notification preferences, account details
Steps:
1. Open the Settings area.
2. Review account details and notification options.
Expected Result:
The page shows account name, email, sync status, and notification controls.

Test 15: Push Notification Support
Requirement: Push notification support
Steps:
1. In Settings, enable notifications.
2. Allow browser notification permission.
3. Click Send Test Notification.
Expected Result:
A browser notification appears.

Test 16: Offline View
Requirement: Offline access using service worker
Steps:
1. Log in and load the dashboard and tasks.
2. Disconnect the internet or switch DevTools to offline mode.
3. Refresh the page or continue viewing cached content.
Expected Result:
Previously loaded task and dashboard data remain available.

Test 17: Offline Task Queue
Requirement: Reliable synchronisation
Steps:
1. While offline, create or update a task.
2. Observe the Pending Sync state.
3. Restore internet connection.
4. Click Sync Now if needed.
Expected Result:
The task change syncs successfully when online.

Requirement: Responsive layout and cross-device compatibility

Steps:
1. Log in as one user.
2. Create tasks and categories.
3. Verify only that user's data is shown.

User data stays isolated and tasks belong only to the authenticated user.
